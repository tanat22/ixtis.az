Security Policy
Reporting a Vulnerability
The ixtis.az project takes the security of its code and website seriously. We appreciate the efforts of security researchers and the community for helping us identify and fix security vulnerabilities.

If you believe you have found a security vulnerability in our project, please let us know immediately. We will work with you to resolve the issue as quickly as possible.

How to Report a Vulnerability
Please follow these steps to report a security vulnerability:

Do not open a public GitHub issue. This may expose the vulnerability to malicious actors before it can be fixed.

Send an email to our security contact at eyunusov@avicom.tech with the following information:

A clear and detailed description of the vulnerability.

The steps to reproduce the vulnerability.

Any potential impact the vulnerability could have.

We will acknowledge receipt of your report within 48 hours and provide a more detailed response within 7 days.

Please give us a reasonable amount of time to fix the vulnerability before making it public. We typically aim to fix critical issues within 30 days.

Scope
This security policy applies to the ixtis.az website and its GitHub repository.

Our Pledge
We are committed to maintaining a secure environment and will do our best to address reported vulnerabilities in a timely and professional manner.

Thank you for helping us keep our project safe.
